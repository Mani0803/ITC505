LIGHTS OUT GAME

Rules of the Game

Clicking on a cell toggles its state and the state of adjacent cells (up, down, left, right).

The goal is to turn off all the lights on the board.

Good luck!

Process Overview:

Understanding the Game Logic:
The first step was to understand the game mechanics. Recognizing how clicking a cell changes its status and neighboring cells was critical.
Structuring the Code:
Organizing the code into separate functions for grid initialization, rendering, switching lights, testing win conditions, and game initiation improved clarity throughout development.
Random Solvable Configuration:
Creating a random, solvable beginning configuration proved tough. The suggestion of imitating clicks rather than randomly selecting individual cells was brilliant. This resulted in a solvable puzzle every time.
Testing and Refinement:
Iterative testing and refinement were necessary. To ensure that the game functioned as expected and that the win condition was correctly tested, extensive testing was required.

Creative Choices:

CSS Styling:
To improve code organization and readability, the "is-off" class was used to represent turned-off lights. CSS classes were also utilized for style.
Starting Configuration:
The use of click simulation to generate a random but solvable starting configuration enabled a fluid and solvable gameplay experience.
Win Condition Alert:
Using window.alert() to display the win condition message and inform the player of their triumph was a simple yet effective choice.

Challenges Faced:

Random Configuration That Ensures Solvability:
Initially, it was challenging to generate a random configuration that ensured solvability. The problem of perhaps building puzzles that couldn't be solved by assigning each cell at random was resolved by simulating clicks.
Ensuring Game Logic Accuracy:
To ensure that there were no logical mistakes and that the toggle mechanism affected nearby cells as anticipated, extensive testing and debugging were required.

Insights Gained:

Algorithmic Thinking:
By considering systematic approaches to solving the puzzle and recognizing how decisions made on a single cell impact the grid as a whole, the game's development enhanced algorithmic thinking.
Code Organization:
Coding was made easier to read, maintain, and debug when it was divided into modular functions.
Problem-Solving Approach:
Having trouble generating a feasible random configuration facilitated the growth of a problem-solving mindset, which in turn facilitated the adoption of a more effective strategy.
